# Flutter Release Readiness Skill

Use this skill before handing off or releasing Flutter work.

## Checklist

- Analysis is clean.
- Relevant tests pass.
- Critical UI states are covered.
- No temporary debug output remains.
- Data migrations or storage assumptions are documented.
- Feature flags or configuration are understood.
- User-facing errors are acceptable.

## Handoff Notes

Include what changed, how it was validated, what remains risky, and what should be monitored after release.
